## Project Overview

Uber Eats operates a large-scale restaurant marketplace where business success depends on factors such as location strategy, pricing, cuisine mix, customer ratings, and platform features like online ordering and table booking.

This project analyses **Uber Eats Bangalore restaurant data** and builds a **Decision Support System** that answers 15 critical business questions using Python and SQL. All results are presented as clean tabular DataFrame outputs in a Streamlit application — no charts or visualisations — mirroring real internal analytics dashboards where stakeholders require precise tabular insights.

## Data Cleaning Steps

The raw Uber Eats dataset required the following cleaning before analysis:

1. **Rate column** — converted from string format `"4.1/5"` to float; non-numeric values like `"NEW"` replaced with `NaN`
2. **Cost column** — removed commas from values like `"1,200"`, converted to integer
3. **Duplicate removal** — dropped 181 duplicate rows
4. **Missing value handling** — rows with null ratings and costs removed
5. **Feature engineering** — created `price_category` (Cheap / Low / Moderate / High / Very High) and `rating_category` (Low / Average / Good) from continuous columns
6. **Column renaming** — cleaned bracket columns (`listed_in(type)` → `listed_type`) for SQL compatibility

## How to Run

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/uber-eats-bangalore.git
cd uber-eats-bangalore
```

### 2. Install dependencies

```bash
pip install streamlit pandas
```

### 3. Set up the database

Run this once to create `food_data.db` from the CSV and JSON files:

```bash
python setup_db.py
```

Expected output:
```
Loading CLEANED1.csv ...
  restaurants table: 23,012 rows loaded
Loading orders.json ...
  orders table: 25,000 rows loaded

Done. Database saved as: food_data.db
Now run:  streamlit run app.py
```

### 4. Launch the Streamlit app

```bash
streamlit run app.py
```

Open your browser at `http://localhost:8501`

## Application Structure

The Streamlit app has **3 pages** accessible from the sidebar:

### Page 1 — Dashboard

- **Quick Analysis** — 4 predefined dropdown queries (Restaurants by Price Category, Average Rating by Location, Rating Segmentation, Average Rating by Price Category)
- **Filter Explorer** — 8 dynamic filters (Location, Restaurant Type, Listed In, Online Order, Table Booking, Price Category, Minimum Rating, Max Cost) that build live SQL `WHERE` clauses
- **Summary metrics** — Total restaurants, Avg Rating, Avg Cost for Two, Avg Votes

### Page 2 — Q&A: Restaurants

15 predefined business questions answered via SQL queries on the `restaurants` table. Each question displays:
- The business value explanation
- A live DataFrame result from SQLite

### Page 3 — Q&A: Orders

10 analytical questions answered via SQL queries on the `orders` table, plus a permanent 5-metric summary strip (Total Orders, Total Revenue, Avg Order Value, Unique Restaurants, Discounted Orders).

## Key Findings

- **Top-rated locations** — Lavelle Road and Koramangala 5th Block consistently lead on average ratings
- **Over-saturated areas** — BTM, Koramangala, and Indiranagar have the highest restaurant density
- **Online ordering** — Restaurants without online ordering have a marginally higher average rating, suggesting premium dine-in restaurants tend to skip the feature
- **Table booking** — Restaurants with table booking show noticeably higher average ratings, confirming it as a premium quality signal
- **Price vs rating** — Higher price categories (High, Very High) correlate with higher ratings, confirming that premium pricing aligns with quality perception
- **Cuisine niche** — Several low-volume cuisine types maintain high ratings, indicating untapped niche potential
